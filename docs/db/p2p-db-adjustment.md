## 1. CREATE TABLE — P2P_TXN_DETAIL

Stores the inquiry snapshot for each P2P gold transfer. A row is created when the customer submits an inquiry (`STATUS = PENDING`) and updated to `COMPLETED` once the transfer is executed.

```sql
CREATE TABLE P2P_TXN_DETAIL (
    TXN_ID          VARCHAR2(50)    NOT NULL,
    CUSTOMER_ID     VARCHAR2(50)    NOT NULL,
    CBS_REF_NO      VARCHAR2(100),
    DR_ACCOUNT_NO   VARCHAR2(50)    NOT NULL,
    DR_ACCOUNT_NAME VARCHAR2(200)   NOT NULL,
    DR_CCY          VARCHAR2(3)     NOT NULL,
    CR_ACCOUNT_NO   VARCHAR2(50)    NOT NULL,
    CR_ACCOUNT_NAME VARCHAR2(200)   NOT NULL,
    CR_CCY          VARCHAR2(3)     NOT NULL,
    GOLD_WEIGHT     NUMBER(12,4)    NOT NULL,
    TOTAL_AMOUNT    NUMBER(18,4)    NOT NULL,
    MEMO            VARCHAR2(500),
    STATUS          VARCHAR2(20)    DEFAULT 'PENDING' NOT NULL,
    CREATED_AT      TIMESTAMP(6)    NOT NULL,
    EXPIRED_AT      TIMESTAMP(6)    NOT NULL,
    CONSTRAINT PK_P2P_TXN_DETAIL PRIMARY KEY (TXN_ID)
);
```

### Column Notes

| Column | Note |
|--------|------|
| `TXN_ID` | Generated by service — format `P2Pyydddsssssmmmrrr` (e.g. `P2P2612370662634398`). Always starts with `P2P`. |
| `CBS_REF_NO` | NULL at `PENDING`; populated with the CBS transaction reference when status moves to `COMPLETED`. |
| `EXPIRED_AT` | Set to `CREATED_AT + 3 minutes`. The service rejects verify requests received after this timestamp. |
| `STATUS` | Only two values: `PENDING` → `COMPLETED`. No other transitions. |

---

## 2. DROP TABLE — P2P_QUOTATION

```sql
DROP TABLE P2P_QUOTATION;
```

`P2P_QUOTATION` was a temporary table used during early development. Its purpose is fully replaced by `P2P_TXN_DETAIL`. No data migration is required — the table contained only development test rows.

---

## 3. P2P_TRANSACTION — No Change

`P2P_TRANSACTION` is unchanged. It continues to record the final completed transfer result inserted by the service after CBS confirms the transfer.

---

## 4. ADD COLUMN — WITHDRAW_TXN.FEE_LIST

```sql
ALTER TABLE WITHDRAW_TXN ADD (
    FEE_LIST VARCHAR2(4000)
);
```

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `FEE_LIST` | VARCHAR2(4000) | NULL | Fee schedule JSON snapshot captured at transaction creation time, keyed by currency code. Preserves the exact fee rules applied to the transaction regardless of future fee configuration changes. |

---

## P2P Flow (for reference)

```
POST /p2p/inquiry
    └── INSERT P2P_TXN_DETAIL  (STATUS = PENDING)

POST /p2p/transfer-quotation-verify
    ├── SELECT P2P_TXN_DETAIL WHERE TXN_ID = :ref  (must be PENDING, not expired)
    ├── INSERT P2P_TRANSACTION  (completed transfer record)
    └── UPDATE P2P_TXN_DETAIL   SET STATUS = 'COMPLETED', CBS_REF_NO = :cbsRefNo
```
