# Table: P2P_TXN_DETAIL

Tracks each P2P gold transfer inquiry initiated by a customer. A row is inserted at inquiry time with `STATUS = PENDING` and updated to `COMPLETED` once the transfer is confirmed by CBS. Acts as the short-lived state store for the 3-minute verification window.

## Columns

| # | Column | Type | Length/Precision | Nullable | Default | Description |
|---|--------|------|-----------------|----------|---------|-------------|
| 1 | TXN_ID | VARCHAR2 | 50 | NOT NULL | | Primary key — generated by service (format: `P2Pyydddsssssmmmrrr`) |
| 2 | CUSTOMER_ID | VARCHAR2 | 50 | NOT NULL | | FK → `CUSTOMER.ID` — the debtor (sender) |
| 3 | CBS_REF_NO | VARCHAR2 | 100 | NULL | | CBS transaction reference returned after transfer; starts with `CBS`; NULL until `STATUS = COMPLETED` |
| 4 | DR_ACCOUNT_NO | VARCHAR2 | 50 | NOT NULL | | Debit (sender) account number |
| 5 | DR_ACCOUNT_NAME | VARCHAR2 | 200 | NOT NULL | | Debit account holder name |
| 6 | DR_CCY | VARCHAR2 | 3 | NOT NULL | | Debit account currency code (e.g. `LBI`) |
| 7 | CR_ACCOUNT_NO | VARCHAR2 | 50 | NOT NULL | | Credit (receiver) account number |
| 8 | CR_ACCOUNT_NAME | VARCHAR2 | 200 | NOT NULL | | Credit account holder name |
| 9 | CR_CCY | VARCHAR2 | 3 | NOT NULL | | Credit account currency code |
| 10 | GOLD_WEIGHT | NUMBER | 12,4 | NOT NULL | | Gold weight to transfer (troy ounces) |
| 11 | TOTAL_AMOUNT | NUMBER | 18,4 | NOT NULL | | Total transfer amount = `GOLD_WEIGHT × sell_rate` at inquiry time |
| 12 | PURPOSE | VARCHAR2 | 500 | NULL | | Optional remark from sender (renamed from `MEMO`) |
| 13 | FEE_CUR | VARCHAR2 | 3 | NULL | | Fee currency code; NULL until `STATUS = COMPLETED` |
| 14 | FEE_AMOUNT | NUMBER | 18,4 | NULL | | Fee amount charged; NULL until `STATUS = COMPLETED` |
| 15 | CR_CBS_SEQNO | VARCHAR2 | 100 | NULL | | CBS credit-side sequence number returned by CBS; NULL until `STATUS = COMPLETED` |
| 16 | DR_CBS_SEQNO | VARCHAR2 | 100 | NULL | | CBS debit-side sequence number returned by CBS; NULL until `STATUS = COMPLETED` |
| 17 | STATUS | VARCHAR2 | 20 | NOT NULL | `'PENDING'` | Lifecycle status — see values below |
| 18 | CREATED_AT | TIMESTAMP(6) | | NOT NULL | | Inquiry creation time |
| 19 | EXPIRED_AT | TIMESTAMP(6) | | NOT NULL | | Expiry time = `CREATED_AT + 3 minutes`; verify is rejected after this |
| 20 | UPDATE_AT | TIMESTAMP(6) | | NULL | | Last update time; set when status transitions to `COMPLETED` |

## STATUS Values

| Value | Meaning |
|-------|---------|
| `PENDING` | Inquiry created — awaiting security question verification |
| `COMPLETED` | Transfer verified and posted to CBS; `CBS_REF_NO`, `DR_CBS_SEQNO`, `CR_CBS_SEQNO`, and `UPDATE_AT` are set |

## TXN_ID Format

Generated by `CommonInfo.genTransactionId("P2P")`:

```
P2P  yy  ddd  sssss  mmm  rrr
 │    │    │     │     │    └── 3-digit random (000–999)
 │    │    │     │     └─────── milliseconds within second (000–999)
 │    │    │     └───────────── second of day (00000–86399)
 │    │    └─────────────────── day of year (001–366)
 │    └──────────────────────── 2-digit year (e.g. 26)
 └───────────────────────────── fixed prefix
```

Example: `P2P2612370662634398`
