# Table: WITHDRAW_TXN

Records every withdraw (transfer-out) transaction initiated by a customer, including debit/credit parties, amounts, fees, provider references, and CBS posting results.

## STATUS Values

| Value | Meaning |
|-------|---------|
| `DEBIT_PENDING` | Initial state — debit not yet posted to CBS |
| `DEBITED` | Debit posted successfully to CBS |
| `CREDIT_PENDING` | Waiting for provider to confirm credit |
| `CREDITED` | Credit confirmed by provider |
| `COMPLETED` | Both debit and credit completed |
| `REVERSAL_PENDING` | Reversal requested, pending execution |
| `REVERSAL` | Reversal completed |
| `REVERSAL_FAILED` | Reversal attempted but failed |
| `FAILED` | Transaction failed (see `ERROR_CODE` / `ERROR_MESSAGE`) |
| `CANCELLED` | Transaction cancelled before processing |
